PROTOCOLS

  • HTTP methods

    GET - use this type of requests to retrieve resource
    representation/information only - and not to modify it in any
    way

    POST - used to create a new resource into the collection of
    resources

    PUT - use it primarily to update existing resource (if the resource br does not exist then API may decide to create a new resource ce or
    not)

    DELETE - used to delete resources (Identifed by Request-URI)

    PATCH - HTTP PATCH requests are to make partial update on a
    resource
  • HTTP status code

    1xx: Informational
    example: 100 Continue

    2xx: Success
    example: 201 Created

    3xx: Redirection
    example: 301 Moved Permanently

    4xx: Client Error
    example: 404 Not Found

    5xx: Server Error
    example: 500 Internal Server Error

  • Identifying google's IP

    To identify google.com IP, follow the next steps:

    1. open Terminal

    2. paste and run the command nslookup google.com

    As result, you will receibe "name" and "address"
    Name: google.com.ua
    Address: 216.58.215.99
    Repeat steps 1 and 2 to identify google.com.ua IP
